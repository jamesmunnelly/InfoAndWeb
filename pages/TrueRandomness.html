Many implementations of public key cryptography depend on randomly generated numbers to ensure their security. This poses the problem that given an attacker can predict the random numbers generated, the security of the system is comprimised.

Truly random values take into account all sorts of data to make it as hard as possible for an attacker to influence the number generated in a predictable manner.

In public key cryptography, a random number generator is used to pick a key. If the random number is predictable, it can become easy to recreate the steps made during initial key generation and computate the private key.

random.org

random.org run a service generating random numbers for anyone who needs them. The numbers are generated from atmospheric noise and so are difficult/near to impossible for an attacker to interfere with it.

Application in RSA encryption

In RSA encryption, the public and private keys should be chosen with adequete entropy, and with a seed length of twice the intended security level (see: Key Size)
